Коментарі 

  Мета включення коментарів в ваш код полягає в тому, щоб пояснити, що робить код.
  Java підтримує як однорядкові, так і багаторядкові коментарі. Всі символи, які знаходяться в коментарі ігноруються компілятором Java.

  Однорядковий коментар починається з двох прямих косих рис і триває до тих пір, поки не досягне кінця рядка.
  Наприклад:
/ / це однорядковий коментар 
x = 5; / / однорядковий коментар після коду
 
Багаторядкові коментарі

  Java підтримує коментарі, які охоплюють кілька рядків.
  Ви починаєте цей тип коментарів з косої риски, за якою слідує зірочка, і закінчуєте його зірочкою, за якою слідує коса риска.
  Наприклад:
/ * це також 
коментар, який охоплює 
кілька рядків * / 

  Зверніть увагу, що Java не підтримує вкладені багаторядкові коментарі.
  Однак можна вкласти однорядкові коментарі в багаторядкові коментарі.
/* це однорядковий коментар

/ / однорядковий коментар 

* / 

Коментарі документації

  Коментарі документації - це спеціальні коментарі, які мають вигляд багаторядкових коментарів, з тією різницею, що вони генерують зовнішню документацію вашого вихідного коду. Вони починаються з косої риски, за якою слідують дві зірочки, і закінчуються зірочкою, за якою слід коса риска.
  Наприклад:
/** це коментар документації */ 

  Javadoc - це інструмент, який знаходиться в JDK, і він використовується для створення документації по коду Java у форматі HTML з вихідного коду Java, який має необхідну документацію у визначеному форматі.

  Коли коментар до документації починається з більш ніж двох зірочок, Javadoc припускає, що ви хочете створити "коробку" навколо коментарів у вихідному коді. Він просто ігнорує додаткові зірочки.
  Наприклад:
/**********************

це і є початок методу

***********************/
